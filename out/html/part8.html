Arcanoid Game Tutorial - Part 8: Sound Effects

Back to Index

In this part of the tutorial, we will add sound effects to the game. Audio feedback makes the gameplay feel more responsive and satisfying. We will create a reusable SoundEffects component and trigger sounds on key gameplay events: paddle hits, wall bounces, brick collisions, life loss, game over, level completion, and ball launch.

Step 1: Prepare Sound Assets

Create a folder assets/sounds/ in your project and place the following .wav files inside:

  • paddle.wav — ball hits the paddle
  • brick_hit.wav — brick is damaged but survives
  • brick_destroy.wav — brick is destroyed
  • wall.wav — ball bounces off a wall or ceiling
  • lose_life.wav — ball falls off screen
  • game_over.wav — last life lost
  • level_complete.wav — all bricks destroyed
  • launch.wav — ball is launched with Space

You can generate retro sound effects for free using tools like sfxr or download them from freesound.org.

Step 2: Create SoundEffects.qml

To keep Main.qml clean, we extract all sound definitions into a separate component. Create a new file qml/SoundEffects.qml:

import Felgo 4.0
import QtQuick 2.0

Item {
    property alias paddle: sfxPaddle
    property alias brickHit: sfxBrickHit
    property alias brickDestroy: sfxBrickDestroy
    property alias wall: sfxWall
    property alias loseLife: sfxLoseLife
    property alias gameOver: sfxGameOver
    property alias levelComplete: sfxLevelComplete
    property alias launch: sfxLaunch

    GameSoundEffect {
        id: sfxPaddle
        source: Qt.resolvedUrl("../assets/sounds/paddle.wav")
    }
    GameSoundEffect {
        id: sfxBrickHit
        source: Qt.resolvedUrl("../assets/sounds/brick_hit.wav")
    }
    GameSoundEffect {
        id: sfxBrickDestroy
        source: Qt.resolvedUrl("../assets/sounds/brick_destroy.wav")
    }
    GameSoundEffect {
        id: sfxWall
        source: Qt.resolvedUrl("../assets/sounds/wall.wav")
    }
    GameSoundEffect {
        id: sfxLoseLife
        source: Qt.resolvedUrl("../assets/sounds/lose_life.wav")
    }
    GameSoundEffect {
        id: sfxGameOver
        source: Qt.resolvedUrl("../assets/sounds/game_over.wav")
    }
    GameSoundEffect {
        id: sfxLevelComplete
        source: Qt.resolvedUrl("../assets/sounds/level_complete.wav")
    }
    GameSoundEffect {
        id: sfxLaunch
        source: Qt.resolvedUrl("../assets/sounds/launch.wav")
    }
}

Each GameSoundEffect loads a single audio file. The property alias declarations expose each sound through a clean public API.

Step 3: Add SoundEffects to Main.qml

In your Scene block inside Main.qml, instantiate the component:

SoundEffects {
    id: sfx
}

All sounds are now accessible as sfx.paddle, sfx.wall, etc.

Step 4: Trigger Sounds on Wall Bounce

In the Timer's onTriggered handler, add sfx.wall.play() when the ball bounces off a wall or the ceiling:

// Wall bounce
if (ball.x <= 0 || ball.x + ball.width >= gameArea.width) {
    ball.ballSpeedX *= -1
    sfx.wall.play()         // <= wall hitting sound
}

if (ball.y <= 0) {
    ball.y = 1
    ball.ballSpeedY *= -1
    sfx.wall.play()         // <= wall hitting sound
}

Step 5: Trigger Sound on Paddle Hit

In the paddle collision block, play the paddle sound:

if (ball.y + ball.height >= paddle.y &&
    ball.x + ball.width >= paddle.x &&
    ball.x <= paddle.x + paddle.width) {
    ball.ballSpeedY *= -1
    sfx.paddle.play()       // <= paddle hitting sound
}

Step 6: Trigger Sounds on Brick Collision

Inside the brick collision loop, play a different sound depending on whether the brick was destroyed or just damaged:

for (let i = 0; i < bricks.count; i++) {
    let item = bricks.itemAt(i)
    if (item && item.collides(ball.x, ball.y, ball.width, ball.height)) {
        let points = item.hit()
        gameArea.score += points
        gameArea.spawnFloatingScore(
            item.x + item.width / 2 - 10,
            item.y,
            points
        )
        ball.ballSpeedY *= -1

        if (item.destroyed)
            sfx.brickDestroy.play()     // <= brick destroyed sound
        else
            sfx.brickHit.play()         // <= brick hit sound

        if (gameArea.checkLevelFinished()) {
            gameArea.levelFinished = true
            ball.ballStuck = true
            sfx.levelComplete.play()        // <= level finished sound
        }

        break
    }
}

Step 7: Trigger Sounds on Life Loss and Game Over

When the ball falls off screen, play either the game-over or lose-life sound:

if (ball.y > gameArea.height) {
    gameArea.lives -= 1
    if (gameArea.lives <= 0) {
        gameArea.lives = 0
        ball.ballStuck = true
        gameArea.gameOver = true
        sfx.gameOver.play()         // <= game over sound
    } else {
        ball.ballStuck = true
        sfx.loseLife.play()         // <= life lost sound
    }
}

Step 8: Trigger Sound on Ball Launch

In the Keys.onPressed handler, play the launch sound when the player presses Space to release the ball:

if (event.key === Qt.Key_Space) {
    if (gameArea.gameOver || gameArea.levelFinished) {
        gameArea.initGame()
    } else if (ball.ballStuck) {
        ball.ballStuck = false
        sfx.launch.play()           // <= launch sound
    }
}

Step 9: Deploy Sound Assets

Make sure the sound files are included in your build. In CMakeLists.txt, add the assets folder to resource deployment:

deploy_resources("assets/sounds")

Summary

We created a dedicated SoundEffects component that encapsulates all audio assets and provides a clean alias-based API. Sounds are triggered at eight key gameplay moments:

SoundEvent
sfx.paddleBall hits the paddle
sfx.wallBall bounces off wall or ceiling
sfx.brickHitBrick is hit but survives
sfx.brickDestroyBrick is destroyed
sfx.loseLifeBall falls off screen (lives remain)
sfx.gameOverLast life is lost
sfx.levelCompleteAll bricks destroyed
sfx.launchBall launched with Space

The game now provides satisfying audio feedback on every interaction.

As the possible next steps, we could think about adding background music or additional levels, as well as various bonus items (like ball slowdown, sticking, extra lives etc.)
