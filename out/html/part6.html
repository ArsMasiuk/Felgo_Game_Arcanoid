Arcanoid Game Tutorial - Part 6: Score Effects

Back to Index

In this part of the tutorial, we will add floating score popups that appear whenever a brick is hit. The text (e.g. "+5", "+10") will drift downward and fade out, giving the player satisfying visual feedback.

Step 1: Create the FloatingScore Component

Create a new file FloatingScore.qml in the qml directory. This component is a Text element that animates itself downward while fading out, then destroys itself automatically.

import QtQuick 2.0

Text {
    id: floatingScore

    property real startX: 0
    property real startY: 0

    x: startX
    y: startY
    color: "yellow"
    font.pixelSize: 14
    font.bold: true

    opacity: 1

    ParallelAnimation {
        id: floatAnimation
        running: false

        NumberAnimation {
            target: floatingScore
            property: "y"
            to: floatingScore.startY + 40
            duration: 600
            easing.type: Easing.OutQuad
        }

        NumberAnimation {
            target: floatingScore
            property: "opacity"
            to: 0
            duration: 600
            easing.type: Easing.InQuad
        }

        onStopped: floatingScore.destroy()
    }

    Component.onCompleted: floatAnimation.start()
}

How it works:

  • startX and startY define where the popup appears (set at creation time).
  • A ParallelAnimation runs two animations simultaneously:
    • The text drifts down by 40 pixels over 600ms (Easing.OutQuad — starts fast, decelerates).
    • The text fades from fully opaque to invisible over 600ms (Easing.InQuad — starts slow, accelerates).
  • When the animation finishes, onStopped calls destroy() to remove the object from memory.
  • The animation starts automatically via Component.onCompleted.

Step 2: Add a Spawn Function to Main.qml

We need a way to dynamically create FloatingScore instances at runtime. Add a Component and a helper function inside gameArea in Main.qml:

// Floating score component
Component {
    id: floatingScoreComponent
    FloatingScore {}
}

function spawnFloatingScore(px, py, points) {
    if (points <= 0) return
    var item = floatingScoreComponent.createObject(gameArea, {
        "text": "+" + points,
        "startX": px,
        "startY": py
    })
}

How it works:

  • floatingScoreComponent is a Component that wraps FloatingScore. This lets us create new instances dynamically using createObject().
  • spawnFloatingScore() takes a position (px, py) and the points value.
  • If points is 0, nothing is spawned (no popup for zero-score hits).
  • createObject() creates a new FloatingScore as a child of gameArea, passing the text and position as initial properties.

Step 3: Spawn Score Popups on Brick Hit

Update the brick collision logic in the Timer to spawn a floating score after each hit. Replace the existing brick collision block with:

// Brick collisions
for (let i = 0; i < bricks.count; i++) {
    let item = bricks.itemAt(i)
    if (item && item.collides(ball.x, ball.y, ball.width, ball.height)) {
        let points = item.hit()
        gameArea.score += points
        gameArea.spawnFloatingScore(
            item.x + item.width / 2 - 10,
            item.y,
            points
        )
        ball.ballSpeedY *= -1
        break
    }
}

Key changes:

  • We store the return value of item.hit() in a local points variable.
  • We pass points to both the score counter and spawnFloatingScore().
  • The popup is positioned at the horizontal center of the brick (offset by 10 pixels to roughly center the text).

Step 4: Run the Game

  1. Build and run the project.
  2. Press Space to launch the ball.
  3. Hit a magenta brick — a yellow "+5" floats down and fades out.
  4. Hit a steelblue brick — a yellow "+10" floats down as the brick is destroyed.
  5. Zero-score hits produce no popup.
