Arcanoid Game Tutorial - Part 1: Basic Game

Back to Index

In this tutorial, we will build the core gameplay of an Arcanoid-style game. We will create a paddle, a bouncing ball, and a grid of bricks to break. The game will be fully playable, and you can extend it with additional features in Part 2.

Prerequisites

Before starting, make sure you have:

  • Felgo SDK installed (includes Qt 6.8+)
  • Basic familiarity with QML syntax
  • A text editor or IDE such as Qt Creator or Visual Studio

Step 1: Create the Game Window

Every Felgo game starts with a GameWindow. This is the top-level element that manages the application window and scaling.

import Felgo 4.0
import QtQuick 2.0

GameWindow {
    id: gameWindow
    activeScene: scene
    screenWidth: 960
    screenHeight: 640
}
  • screenWidth and screenHeight set the actual window size in pixels.
  • activeScene tells Felgo which Scene to display.

Step 2: Add a Scene

Inside the GameWindow, add a Scene. The scene defines a logical size that auto-scales to fit the window.

Scene {
    id: scene
    width: 480
    height: 320

    focus: true
    Component.onCompleted: forceActiveFocus()
}
  • The logical size is 480×320. Felgo scales this to the actual window size.
  • focus: true and forceActiveFocus() are required to receive keyboard input.

Step 3: Set Up the Game Area

Add a black Rectangle that fills the scene. This acts as the game area and holds the score.

Rectangle {
    id: gameArea
    anchors.fill: parent
    color: "black"

    property int score: 0
}
PropertyPurpose
scoreTracks the player's current score

Step 4: Create the Paddle

The paddle is a simple white rounded rectangle near the bottom of the screen.

Rectangle {
    id: paddle
    width: 100
    height: 15
    color: "white"
    radius: 5
    y: parent.height - 30
    x: parent.width / 2 - width / 2
}

The paddle starts centered horizontally. It will be moved by keyboard input (handled in Step 7).

To make the paddle movable, lets extend paddle by adding the following properties to it:

Rectangle {
    id: paddle
    ...

    property real paddleSpeed: 6
    property bool leftPressed: false
    property bool rightPressed: false
}
PropertyPurpose
leftPressedLeft button has been pressed -> paddle moves to the left
rightPressedRight button has been pressed -> paddle moves to the right
paddleSpeedHow many pixels the paddle moves per frame

Step 5: Create the Ball

The ball is a small red circle.

Rectangle {
    id: ball
    width: 12
    height: 12
    radius: 6
    color: "red"
    x: parent.width / 2
    y: parent.height / 2
}

Note: In QML, you create a circle by setting radius to half of width and height.

Lets add some basic properties to the ball as well:

Rectangle {
    id: ball

    ...
    property real ballSpeedX: 3
    property real ballSpeedY: 3
    property bool ballStuck: true
}
PropertyPurpose
ballSpeedX, ballSpeedYBall velocity in pixels per frame
ballStuckWhen true, the ball stays attached to the paddle

Step 6: Build the Brick Grid

Use a Repeater to generate a grid of bricks. The brick layout properties are defined on the Repeater itself, and each brick calculates its own column and row from its index.

Repeater {
    id: bricks
    model: bricks.cols * bricks.rows

    property int cols: 10
    property int rows: 3
    property real brickSpacing: 4
    property real brickWidth: (gameArea.width - (cols + 1) * brickSpacing) / cols
    property real brickHeight: 20

    Rectangle {
        width: bricks.brickWidth
        height: bricks.brickHeight
        color: "steelblue"
        border.color: "white"

        property int col: index % bricks.cols
        property int row: Math.floor(index / bricks.cols)

        x: bricks.brickSpacing + col * (bricks.brickWidth + bricks.brickSpacing)
        y: bricks.brickSpacing + row * (bricks.brickHeight + bricks.brickSpacing)

        property bool destroyed: false
        visible: !destroyed
    }
}

How the grid works:

  • The layout properties (cols, rows, brickSpacing, brickWidth, brickHeight) live on the bricks Repeater.
  • The model produces cols × rows items (10 × 3 = 30 bricks).
  • Each brick computes its col and row from the flat index.
  • The x and y positions include spacing between bricks.
  • brickWidth is calculated from gameArea.width to fill the available space evenly.
  • When destroyed is set to true, the brick becomes invisible.

Step 7: Handle Keyboard Input

Add key handlers to the Scene to control the paddle and launch the ball.

Keys.onPressed: function(event) {
    if (event.key === Qt.Key_Left)
        paddle.leftPressed = true

    if (event.key === Qt.Key_Right)
        paddle.rightPressed = true

    if (event.key === Qt.Key_Space && ball.ballStuck) {
        ball.ballStuck = false
        event.accepted = true
    }
}

Keys.onReleased: function(event) {
    if (event.key === Qt.Key_Left)
        paddle.leftPressed = false

    if (event.key === Qt.Key_Right)
        paddle.rightPressed = false
}
  • Left/Right arrows set boolean flags that the game loop reads each frame.
  • Space launches the ball by setting ballStuck to false.

Step 8: Implement the Game Loop

The game loop is a Timer that fires every 16 ms (~60 FPS). It handles all movement and collision logic.

Paddle Movement

if (paddle.leftPressed)
    paddle.x -= paddle.paddleSpeed

if (paddle.rightPressed)
    paddle.x += paddle.paddleSpeed

// Clamp inside screen
if (paddle.x < 0)
    paddle.x = 0

if (paddle.x + paddle.width > gameArea.width)
    paddle.x = gameArea.width - paddle.width

The paddle moves based on the key flags and is clamped so it cannot leave the screen.

Ball Movement (Stuck Mode)

if (ball.ballStuck) {
    ball.x = paddle.x + paddle.width / 2 - ball.width / 2
    ball.y = paddle.y - ball.height - 2
    return
}

While stuck, the ball sits centered on top of the paddle. Pressing Space releases it.

Ball Movement (Free Mode)

ball.x += ball.ballSpeedX
ball.y += ball.ballSpeedY

Each frame, the ball moves by its speed values in both axes.

Step 9: Add Collision Detection

Ball vs. Walls

if (ball.x <= 0 || ball.x + ball.width >= gameArea.width)
    ball.ballSpeedX *= -1

if (ball.y <= 0) {
    ball.y = 1
    ball.ballSpeedY *= -1
}

The ball bounces off the left, right, and top walls by reversing the appropriate speed component.

Ball vs. Paddle

if (ball.y + ball.height >= paddle.y &&
    ball.x + ball.width >= paddle.x &&
    ball.x <= paddle.x + paddle.width) {
    ball.ballSpeedY *= -1
}

This is a simple AABB (axis-aligned bounding box) collision check. When the ball overlaps the paddle, its vertical direction is reversed.

Ball vs. Bottom Edge

if (ball.y > gameArea.height) {
    ball.ballStuck = true
}

If the ball falls below the screen, it reattaches to the paddle (like losing a life).

Ball vs. Bricks

for (let i = 0; i < gameArea.children.length; i++) {
    let item = gameArea.children[i]
    if (item.destroyed !== undefined && !item.destroyed) {
        if (ball.x < item.x + item.width &&
            ball.x + ball.width > item.x &&
            ball.y < item.y + item.height &&
            ball.y + ball.height > item.y) {

            item.destroyed = true
            ball.ballSpeedY *= -1
            break
        }
    }
}

The loop iterates over all children of gameArea. It identifies bricks by checking for the destroyed property. On collision:

  1. The brick is marked as destroyed (becomes invisible).
  2. The ball's vertical speed is reversed.
  3. The loop breaks so only one brick is destroyed per frame.

Step 10: Run the Game

  1. Build and run the project.
  2. Press Space to launch the ball.
  3. Use Left and Right arrow keys to move the paddle.
  4. Destroy all bricks to win!

Next Steps

Here are some ideas to extend the game:

  • Add a score counter using the existing gameArea.score property and a Text element.
  • Add lives — track how many times the ball falls and show a game-over screen.
  • Add colored brick rows with different point values.
  • Implement angle-based bouncing — change ballSpeedX based on where the ball hits the paddle.
  • Add sound effects using the Felgo GameSoundEffect type.
  • Add a win condition that detects when all bricks are destroyed.
